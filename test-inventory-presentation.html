Unit Test Gaps

asp-fmc: Zero unit tests, zero code coverage
First Mutable Code -- the root of trust. All validation relies on SimNow boot + static analysis. Critical paths (DICE attestation, A/B recovery, crypto-seed verification) have no isolated tests.
nht-firmware: Zero unit tests
Only SimNow boot serves as functional gate. No test for messaging interface, hotplug event handling, or PAL abstraction layer.
sriov-dr: Zero unit tests, zero SimNow, zero functional tests
SR-IOV virtualization driver has only static analysis (Coverity + lint). No test for VF creation, partition logic, or virtualization features.
amd-tee3_0: TA unit tests exist but are Windows-only (VS project)
Not integrated into CI. DDK test suites in amd_tee_test/test_suites/ also not run in any workflow.
art-security: CONFIG_TEST variants require SimNow to execute
12 test files under common/test/ (ART_LOG, CALIPTRA, DPE, RAP, RSMU, SPDM, STARTUP, WAFL, FUSE) -- no host-side unit test framework.
mpifoe-fw: Twister + pytest tests unclear if in PR gate
testcase.yaml for RAS and telemetry exist. pytest regressions exist. Jenkins pipelines exist. But GHA integration is incomplete.

Top 7 Ranked Gaps

1. cp-mi400 and sw-security-tools: Zero CI
No workflows, no static analysis, no tests, no build verification. cp-mi400 is GPU command processor microcode. sw-security-tools contains security-critical signing tools. Both can ship broken code with no automated detection.
2. pmfw-firmware: Minimal CI with 5 PSP entries
Produces 5 PSP entries (MP1, MP5, IMU instruction + data) for power management. No lint, no SimNow in PR, test infra exists but not integrated. Coverity recently added (3x builds) but no nightly.
3. Only 5 of 16 repos run unit tests in PR gate
dcgpu-esid (CppUTest), mpio (custom x86), VBL-TEE-Drv (shared CppUTest), pcie-cld-fw (GoogleTest), ucie-fw (GoogleTest). Remaining 11 repos have no host-side unit test execution in CI.
4. MPRAS-Applets: No SimNow, no nightly, no functional test
A broken applet can merge with zero boot validation. Entirely depends on MPRAS-Kernel pipeline. PR gate only builds pcie_applet for WH SP7.
5. sriov-dr: CI exists but zero functional tests
Has Coverity + lint + build in PR, and BlackDuck in nightly. But no unit tests, no SimNow, no emulation. Only static analysis validates the code.
6. No code coverage measurement in any repo (automated)
ucie-fw has gcov in nightly; pcie-cld-fw has Bullseye manual-only; art-security has Coco (manual). No repo has automated coverage in PR gate or nightly reporting.
7. No cross-repo integration testing
Each repo tests its own binary in isolation via IFWI merge. No pipeline validates all components together in a single IFWI. No regression tracking or test result trending.

What We Found

  • The top 4 repos (art-security, asp-fmc, ucie-fw, dcgpu-esid) demonstrate that comprehensive CI is achievable with this infrastructure
  • Shared IVV workflows (PFO/ivv-workflows) provide a powerful pattern -- repos using them get Coverity, lint, SimNow, and AI review with minimal per-repo configuration
  • The biggest systemic gap is host-side unit testing: only 5 of 16 repos run unit tests in PR gate
  • The biggest risk is the 2 repos with zero CI: security-critical tools and GPU command processor microcode shipping unvalidated
  • 3 HW test systems exist (CI-Lite ~56 min, DCAuto ~62 min, SimOxide pre-si) but are not uniformly wired into all repos

What To Do Next

  • Immediate: Onboard cp-mi400 and sw-security-tools onto basic CI (build + lint + Coverity)
  • Short-term: Add lint to pmfw-firmware. Wire SimNow into pmfw and MPRAS-Applets PR gates. Add nightly workflows to the 6 repos missing them
  • Medium-term: Standardize on CppUTest (PFO) / GoogleTest (NBIOFW) for host-side unit tests. Add AI code review to all repos via existing IVV workflow
  • Long-term: Implement uttr-tests.yaml schema across all 16 components. Build Merlin AI test acceptance with golden baselines. Enable cross-component integration testing

All numbers in this presentation are derived from actual workflow files, test directories, and SimNow configs across the 16 analyzed repositories.
