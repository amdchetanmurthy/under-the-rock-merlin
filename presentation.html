AI Test Generation — Without Hallucination

We don't ask owners to write tests. We observe what known-good firmware does.

The Pipeline

1. Collect — Boot LKG firmware in SimNow, record every register value, log marker, and timing
2. Generate — AI creates acceptance tests from these observations
3. Ground — Validate every assertion against real data. Ungrounded = advisory only
4. Execute — Run grounded assertions against PR's SimNow boot
5. Learn — After 5 consistent observations, new assertions become grounded

SimNow — Our Test Engine

AMD's pre-silicon simulation validates firmware without physical hardware

What SimNow gives us

  • Boot IFWI images in simulated MI450 hardware
  • 20 pre-defined MI450 test configurations
  • Secure and non-secure boot modes
  • Multiple die configurations (M112, M222, M228, A+A)
  • FW status register monitoring at boot milestones
  • Deterministic, reproducible test results

Pass criteria

MPASP_FW_STATUS = 0x800F0000

All ESIDs completed · TOS steady state · Error log empty

Merlin's SimNow strategy

Version Pinned

SimNow nightly builds can break compatibility. We pin to a validated version and update only after regression-testing the new version.

Model Selection

IFoE changes → 2G IFoE model. RAS changes → MBAT config. Basic boot → M112 (fastest). Full system → A+A.

Non-Secure for CI

CI uses test-mode fuses (unsigned firmware accepted). No signing required for pre-merge validation.
