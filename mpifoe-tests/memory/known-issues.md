## Format

Each entry follows this structure:

```
### <JIRA-ID>: <Short Description>
- **Status**: open | resolved | workaround
- **Affected tests**: <list of test case paths>
- **Symptoms**: <what the failure looks like>
- **Root cause**: <known or suspected cause>
- **Workaround**: <steps to mitigate>
- **Resolution**: <fix details, if resolved>
- **Added**: <date>
- **Last seen**: <date>
```
